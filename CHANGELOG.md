# Changelog

## HTH Framework v0.6 — 2026-07-18

### Added

- Canonical stage banners with `HTH :: STAGE_*` identity.
- UTC start and completion timestamps for every current pipeline stage.
- Per-stage elapsed-time JSONL records and Pipeline Health timing table.
- Pipeline start and summary-generation timestamps.
- Publication Manifest and Pipeline Health documentation.
- Architecture and development guides.
- Top-level project overview and maintainer contact information.

### Changed

- Standardized publication terminology in workflow-facing reports.
- Clarified separation between framework v0.6 and AutoHotkey capture v0.5.
- Elevated documentation to a versioned part of the pipeline architecture.

## Earlier development

Before v0.6, HTH established preprocessing, physical-page analysis, geometry
candidate detection, reference validation, reusable workflows, detector
registration and resilience, and production/test publication plumbing.
